# Regulatory AI in Oil and Gas

Interactive framework for qualifying Agentic AI in business processes for Oil & Gas operations in Norway.

## Overview

This framework maps the full qualification lifecycle for deploying AI agents in a regulated Oil & Gas environment, covering:

- **Regulatory Landscape** — PSA, EU AI Act, NORSOK/ISO, GDPR/NIS2
- **Technology Qualification** — DNV-RP-A203 adapted for AI/ML systems
- **Agent Qualification Pipeline** — Per use-case gate review process
- **Audit Trail Architecture** — Immutable, defensible evidence chain
- **Runtime Governance** — Monitoring, guardrails, drift detection
- **Use-Case Classification** — Risk-based regulatory mapping
- **Evidence Lifecycle** — End-to-end defensibility assurance

## Usage

Open `index.html` in a browser. The interface supports:

- **Add/Remove** process components via the UI
- **Collapse/Expand** sections for focused views
- **Export/Import** the framework as JSON for version control
- Edit the HTML directly to customize sections, regulations, or use-cases

## Key Regulations Covered

| Regulation | Scope |
|---|---|
| PSA (Petroleumstilsynet) | Safety, barrier management, MTO |
| EU AI Act | Risk classification, conformity assessment |
| NORSOK Z-013 / S-001 | Risk assessment, technical safety |
| DNV-RP-A203 | Technology qualification |
| IEC 61508 | Functional safety |
| GDPR | Personal data processing |
| NIS2 / DORA | Cybersecurity, digital resilience |
