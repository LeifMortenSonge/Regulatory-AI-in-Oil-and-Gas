SHapley Additive exPlanations for Defensible AI in Oil & Gas (Norway)

1 — What SHAP Provides & Why It Matters Here Foundation

SHAP decomposes any AI prediction into per-feature contribution values grounded in Shapley values from cooperative game theory. In a regulated Oil & Gas context, this transforms opaque agent decisions into auditable, quantifiable evidence.

🧮 Mathematical Foundation

For each prediction, SHAP computes the marginal contribution of every feature, satisfying three axioms: local accuracy (values sum to the prediction), missingness (absent features get zero), and consistency (monotone feature importance).

Shapley Values Game Theory Additive

⚖️ Regulatory Defensibility

Unlike attention weights or saliency maps, SHAP values have a proven theoretical basis — making them defensible under PSA inspection, EU AI Act conformity assessment (Art. 13-14), and DNV-RP-A203 qualification.

EU AI Act Art. 13 PSA §§ 4-5 DNV-RP-A203

🔌 Model-Agnostic

Works with any model: XGBoost, neural networks, LLM-based classifiers, ensemble agents. The same explanation framework applies whether your agent uses a gradient-boosted tree or a fine-tuned transformer.

TreeSHAP KernelSHAP DeepSHAP
Key principle: Every agent decision in a safety-critical Oil & Gas process must answer "why?" with quantifiable evidence. SHAP provides that answer in a form that auditors, regulators, and courts can evaluate mathematically.

3 — Technical Implementation — Code Patterns Implementation

Core pattern: wrap your agent's decision model with a SHAP explainer. Works for tabular models (sensor data, operational parameters) and structured LLM outputs.

import shap import xgboost as xgb import numpy as np from dataclasses import dataclass, field from datetime import datetime, timezone # ── 1. Train or load the agent's decision model ── # Example: Well integrity risk prediction from sensor data model = xgb.XGBClassifier() model.load_model("models/well_integrity_v3.json") # ── 2. Create SHAP explainer with background dataset ── # Background data = representative sample of normal operations background = shap.utils.sample(X_train, nsamples=100) explainer = shap.TreeExplainer(model, data=background) # ── 3. Explain a single agent decision ── def explain_decision(features: np.ndarray, feature_names: list) -> dict: """Generate SHAP explanation for one agent decision.""" shap_values = explainer.shap_values(features) # Build per-feature attribution attributions = { name: float(val) for name, val in zip(feature_names, shap_values[0]) } # Sort by absolute impact ranked = sorted( attributions.items(), key=lambda x: abs(x[1]), reverse=True ) return { "base_value": float(explainer.expected_value), "prediction": float(model.predict_proba(features)[0, 1]), "attributions": attributions, "top_drivers": ranked[:5], "timestamp": datetime.now(timezone.utc).isoformat(), }

SHAP middleware intercepts every agent decision, generates explanations, and attaches them to the decision record before it reaches the audit trail.

from functools import wraps from typing import Callable, Any import hashlib, json class SHAPMiddleware: """Intercept agent decisions, compute SHAP, attach to audit trail.""" def __init__(self, explainer, feature_names, audit_store, guardrail_engine): self.explainer = explainer self.feature_names = feature_names self.audit_store = audit_store self.guardrails = guardrail_engine def wrap_agent_action(self, agent_func: Callable) -> Callable: """Decorator: explain every agent decision automatically.""" @wraps(agent_func) def wrapper(features, *args, **kwargs): # 1. Run the agent's decision decision = agent_func(features, *args, **kwargs) # 2. Compute SHAP explanation explanation = explain_decision(features, self.feature_names) # 3. Run guardrail checks against SHAP values guardrail_result = self.guardrails.check(explanation) # 4. Build immutable audit record record = { "decision": decision, "explanation": explanation, "guardrails": guardrail_result, "features_hash": hashlib.sha256( features.tobytes() ).hexdigest(), } # 5. Store in immutable audit log self.audit_store.append(record) # 6. If guardrails failed → escalate to human if not guardrail_result["passed"]: return self._escalate(record) return decision return wrapper

SHAP-based guardrails enforce regulatory boundaries deterministically. If a safety-critical feature's SHAP value falls outside the validated envelope, the decision is blocked — not merely flagged.

from dataclasses import dataclass from typing import Dict, List, Optional @dataclass class SHAPGuardrail: """One guardrail rule tied to SHAP attribution boundaries.""" feature: str min_shap: Optional[float] = None # min expected SHAP contribution max_shap: Optional[float] = None # max expected SHAP contribution must_be_top_n: Optional[int] = None # must rank in top N drivers regulation: str = "" # regulatory reference action: str = "block" # block | escalate | warn class SHAPGuardrailEngine: """Deterministic guardrails based on SHAP attribution envelopes.""" def __init__(self, rules: List[SHAPGuardrail]): self.rules = rules def check(self, explanation: dict) -> dict: violations = [] for rule in self.rules: val = explanation["attributions"].get(rule.feature, 0.0) rank = self._get_rank(rule.feature, explanation) if rule.min_shap is not None and val < rule.min_shap: violations.append({ "rule": f"{rule.feature} SHAP below minimum", "expected": f"≥ {rule.min_shap}", "actual": val, "regulation": rule.regulation, "action": rule.action, }) if rule.must_be_top_n and rank > rule.must_be_top_n: violations.append({ "rule": f"{rule.feature} not in top {rule.must_be_top_n} drivers", "rank": rank, "regulation": rule.regulation, "action": rule.action, }) return { "passed": len(violations) == 0, "violations": violations, "blocking": any(v["action"] == "block" for v in violations), } # ── Example: Well Integrity Guardrails ── well_guardrails = SHAPGuardrailEngine([ SHAPGuardrail( feature="annulus_pressure", must_be_top_n=3, regulation="PSA Facilities Reg. § 48 — Barrier monitoring", action="block", ), SHAPGuardrail( feature="casing_integrity_score", min_shap=0.05, regulation="NORSOK D-010 — Well barrier requirements", action="block", ), SHAPGuardrail( feature="temperature_deviation", max_shap=0.4, regulation="IEC 61508 — Functional safety bounds", action="escalate", ), ])

Monitor SHAP value distributions over time. When the features driving decisions shift, that's an explanation drift signal — triggering re-qualification before output quality degrades.

import numpy as np from scipy import stats from typing import Dict, Tuple class SHAPDriftDetector: """Detect explanation drift using SHAP value distributions.""" def __init__(self, baseline_shap_values: np.ndarray, feature_names: list, alpha=0.01): self.baseline = baseline_shap_values # shape: (n_samples, n_features) self.feature_names = feature_names self.alpha = alpha # significance level for drift test def detect_drift(self, recent_shap_values: np.ndarray) -> Dict: """Compare recent SHAP distributions against baseline.""" drift_report = {"drifted_features": [], "details": {}} for i, name in enumerate(self.feature_names): baseline_col = self.baseline[:, i] recent_col = recent_shap_values[:, i] # Kolmogorov-Smirnov test: are distributions different? ks_stat, p_value = stats.ks_2samp(baseline_col, recent_col) # Mean shift in SHAP importance mean_shift = abs(recent_col.mean() - baseline_col.mean()) # Rank change detection baseline_rank = self._importance_rank(self.baseline, i) recent_rank = self._importance_rank(recent_shap_values, i) rank_change = recent_rank - baseline_rank is_drifted = p_value < self.alpha drift_report["details"][name] = { "ks_statistic": float(ks_stat), "p_value": float(p_value), "mean_shift": float(mean_shift), "rank_change": int(rank_change), "drifted": is_drifted, } if is_drifted: drift_report["drifted_features"].append(name) drift_report["overall_drift"] = len( drift_report["drifted_features"] ) > 0 drift_report["severity"] = self._classify_severity(drift_report) return drift_report def _classify_severity(self, report) -> str: n = len(report["drifted_features"]) if n == 0: return "none" if n <= 2: return "low" # monitor if n <= 5: return "medium" # investigate return "high" # trigger re-qualification

Every decision produces a tamper-evident audit record with SHAP explanation, guardrail results, and cryptographic hash chain — meeting PSA evidence requirements and EU AI Act Art. 12 record-keeping.

import hashlib, json from datetime import datetime, timezone from dataclasses import dataclass, asdict from typing import Dict, Any, Optional @dataclass class SHAPAuditRecord: """Immutable, hash-chained audit record for one agent decision.""" # Identity record_id: str agent_id: str use_case: str # e.g., "well-integrity-assessment" timestamp: str # ISO 8601 UTC # Decision decision: Any # the agent's output confidence: float # model confidence score # SHAP Explanation shap_base_value: float # expected value (population average) shap_attributions: Dict[str, float] # feature → SHAP value shap_top_drivers: list # top-N features by |SHAP| # Guardrails guardrail_passed: bool guardrail_violations: list human_escalated: bool # Regulatory mapping applicable_regulations: list # ["PSA §48", "EU AI Act Art.13"] risk_level: str # high / medium / low # Integrity input_hash: str # SHA-256 of input features model_version: str # model artifact identifier previous_record_hash: str # chain to previous record record_hash: str = "" # computed on creation def __post_init__(self): # Compute tamper-evident hash of this entire record payload = json.dumps(asdict(self), sort_keys=True, default=str) self.record_hash = hashlib.sha256( payload.encode() ).hexdigest() # ── Store to WORM-compliant immutable storage ── def store_audit_record(record: SHAPAuditRecord, store): """Persist to tamper-proof storage (Azure Blob WORM / S3 Object Lock).""" payload = json.dumps(asdict(record), indent=2, default=str) store.write_immutable( key=f"audit/{record.use_case}/{record.record_id}.json", data=payload, content_hash=record.record_hash, )

7 — SHAP Explainer Selection Guide Reference

Choose the right SHAP explainer based on your agent's model type and latency requirements.

ExplainerModel TypesSpeedExact?Oil & Gas Use CasePython Class
TreeSHAP XGBoost, LightGBM, Random Forest, CatBoost Fast (O(TLD²)) Yes Well integrity, maintenance, emissions (tabular sensor data) shap.TreeExplainer
KernelSHAP Any model (model-agnostic) Medium (sampling-based) Approximate LLM-based agents, ensemble models, black-box APIs shap.KernelExplainer
DeepSHAP Neural networks (PyTorch, TensorFlow) Medium Approximate Drilling advisor (LSTM), anomaly detection (autoencoder) shap.DeepExplainer
LinearSHAP Linear / logistic regression Very Fast Yes Compliance scoring, risk classification (interpretable baselines) shap.LinearExplainer
PermutationSHAP Any model (model-agnostic) Slow Approximate Validation / cross-check of primary explainer results shap.PermutationExplainer
Recommendation for real-time agents: Use TreeSHAP with tree-based models for sub-millisecond explanations on tabular sensor data. For LLM-based agents, extract structured features first, then apply KernelSHAP on the structured representation. Reserve DeepSHAP for neural network models on streaming time-series (drilling, process control).

  • Storage: Azure Blob (immutable policy) for WORM audit
  • Database: TimescaleDB on Azure for SHAP time-series
  • Queue: Azure Service Bus for async explanation jobs

Azure Kubernetes Managed Services

🔒 Security & Access

  • RBAC: Separate roles for SHAP config vs. read-only audit
  • Encryption: TLS in transit, AES-256 at rest
  • Network: Private endpoints, no public SHAP API exposure
  • Audit: Azure Monitor + Sentinel for access logging
Sikkerhetsloven NIS2 Zero Trust

📈 Scaling Strategy

  • Real-time: TreeSHAP in-process (~0.1ms per decision)
  • Near-real-time: KernelSHAP via Celery workers (~1-5s)
  • Batch: Nightly SHAP drift analysis on GPU nodes
  • Caching: Redis for repeated feature pattern explanations
Horizontal Scaling GPU Optional Edge-Compatible
Edge deployment note: For offshore platforms with limited connectivity, TreeSHAP can run locally on edge hardware. SHAP records are queued and synced to cloud WORM storage when connectivity is restored — maintaining audit completeness even during communication blackouts.
